# NexVoice 二次 AI 转写方案

更新时间：2026-06-19

## 目标

腾讯云 ASR 负责把语音尽快转成文字；二次 AI 转写负责把这段原始口语稿整理成可直接发送、记录或粘贴的文本。

当前已按“实时识别”和“最终润色”分离的方式接入：

1. 录音中：只展示腾讯云实时识别草稿，不写入输入框，也不做 AI 改写。
2. 录音结束：拿到腾讯云 final 文本后调用文本模型润色。
3. 润色成功：把润色后的文本写入当前输入框。
4. 润色失败或超时：回退写入原始 ASR final 文本，不能让用户丢内容。

## 推荐方案

推荐先做“最终稿润色”，暂不做“边说边 AI 改写”。

优点：

- 体验稳定：输入框只写一次，不会来回跳字。
- 成本可控：每次录音只调用一次文本模型。
- 容易回退：AI 失败时直接写原始识别结果。
- 易验收：可以分别量化 ASR 延迟、AI 润色耗时、总写入耗时。

缺点：

- 用户停止说话后会多等一次 AI 调用。
- 如果文本很长，需要做超时和长度控制。

## 不推荐第一版做的方案

### 边识别边 AI 改写

优点是看起来更智能；缺点是 partial 文本会不断变化，AI 输出也会变化，容易闪烁、成本高、错误多。第一版不建议。

### 先写原文，再替换成润色稿

优点是用户马上看到文字；缺点是需要在任意 App 里可靠选中并替换刚插入的文字，跨 App 风险高。第一版不建议。

## 模型建议

DeepSeek 官方 API 当前兼容 OpenAI Chat Completions 格式，base URL 为 `https://api.deepseek.com`。截至 2026-06-19，官方文档展示的新模型名是 `deepseek-v4-flash` / `deepseek-v4-pro`，并提示 `deepseek-chat` 将在 2026-07-24 15:59 UTC 弃用。

第一版建议：

- 默认模型：`deepseek-v4-flash`
- 模式：关闭 thinking / reasoning，避免额外延迟
- 超时：5 秒
- 失败策略：回退原始 ASR final 文本

## 推荐提示词方向

系统提示词应限制模型只做整理，不改变事实：

```text
你是语音输入文本整理器。请把用户的口语转写整理成自然、清晰、可直接发送的文本。
要求：保留原意；修正明显错别字和标点；删除口头禅、重复词和无意义停顿；不要新增事实；不要解释；只输出整理后的文本。
```

后续可以按场景加模式：

- 默认：自然、清晰、轻度润色。
- 聊天：口语自然，不要太正式。
- 文档：更完整、分段、标点清楚。
- 翻译：保留原意，目标语言自然。

## 当前落地状态

已完成：

- 新增 `DeepSeekFinalRewriteService`。
- 新增 `DeepSeekFinalRewriteConfiguration` 和本机私有配置读取。
- 新增 `scripts/configure_deepseek.sh` 写入 `~/Library/Application Support/NexVoice/DeepSeek.json`。
- 在腾讯云 `finalTranscript` 到达后进入 DeepSeek 整理。
- DeepSeek 成功后插入整理文本。
- DeepSeek 失败或超时后插入原始 ASR final 文本。
- 用户开始下一次录音或按 ESC 取消当前会话时，会取消上一轮未完成的整理任务，避免旧文字被插入。
- 当前默认提示词已从“单纯润色”调整为“整理思路”：保留原意，不新增事实；遇到多个要求、步骤、问题、原因或结论时，自动整理成编号或清晰分段。
- 当前支持输出语言模式：`输出：中文` 以中文为主并允许保留英文术语和自然中英混合；`输出：English` 会把最终文本整理为英文。
- 当前支持单一 `输出模式` 二级菜单：`标准模式（默认）`、`社交达人`、`强化嘴替`、`冷静模式`。语言和输出模式相互独立；输出模式同时控制改写幅度和表达调性，避免拆成多个用户难以理解的开关。
