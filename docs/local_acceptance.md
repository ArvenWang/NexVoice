# NexVoice 本地验收说明

更新时间：2026-06-21

## 当前可验收范围

本轮可验收的是腾讯云实时 ASR + DeepSeek 最终整理语音输入链路，以及第一版看屏自动回复链路：

1. 打开 NexVoice 菜单栏 App。
2. 申请麦克风权限。
3. 申请辅助功能权限。
4. 申请屏幕录制权限。
5. 选择最终输出语言：中文或英文；识别阶段不需要手动选择输入语言。
6. 打开任意可输入文字的位置，例如备忘录、浏览器搜索框、聊天输入框。
7. 按一下默认右 Alt 快捷键开始语音输入。
8. 在屏幕底部看到自动叠加的统一浮层，浮层内包含实时识别草稿和小型波形。
9. 再按一下右 Alt 结束，统一浮层应缩小为 loading 条；腾讯云返回最终结果后，App 会结合当前 App、输入框上下文、输出模式和个人词库调用 DeepSeek 做最终整理，成功后把整理后的文本输入当前聚焦的输入框。
10. 在聊天、邮件或网页输入框中长按右 Alt，触发看屏回复；App 应识别当前前台窗口可见文字，按当前输出模式生成一条回复并写入当前输入框，但不自动发送。
11. 在设置里自定义快捷键并再次触发语音输入和看屏回复。

当前还没有实时同传。腾讯云实时 ASR 需要先配置 AppID、SecretId 和 SecretKey；DeepSeek 最终整理需要配置 API Key；SenseVoice 本地后端保留为兜底和对照。

可选个人词库文件位于：

```text
~/Library/Application Support/NexVoice/PersonalDictionary.json
```

文件不存在时不影响使用。需要提升专有名词识别时，可以写入：

```json
{
  "terms": [
    {"phrase": "NexVoice", "weight": 11, "note": "产品名"},
    {"phrase": "DeepSeek", "weight": 9}
  ]
}
```

## 启动方式

在项目根目录运行：

```bash
./scripts/configure_tencent_asr.sh
./scripts/configure_deepseek.sh
./scripts/dev_run.sh
```

脚本会重新构建并打开：

```text
dist/NexVoice.app
```

打开后，顶部菜单栏会出现 `NexVoice`。

## 验收步骤

1. 点击菜单栏 `NexVoice`。
2. 点击 `申请麦克风权限`，在系统弹窗中允许。
3. 点击 `申请辅助功能权限`，在系统弹窗或系统设置中允许 NexVoice 控制电脑；这是接收全局键盘事件和把文字输入到其他 App 的必要权限。
4. 如果系统设置列表里看不到 NexVoice，先回到菜单再次点击 `申请辅助功能权限`，让 macOS 先登记这个 App。
5. 点击 `申请屏幕录制权限`，在系统设置里允许 NexVoice 录制屏幕；这是看屏回复读取当前窗口可见文字的必要权限。
6. 如果 macOS 提示需要重新打开 App，退出 NexVoice 后重新运行 `./scripts/dev_run.sh`。
7. 根据最终想输出的语言点击 `输出：中文` 或 `输出：English`；说话时可以说中文、英文或中英混合。
8. 打开一个文本输入框，并让光标停在里面。
9. 按一下键盘右 Alt 并说一段中文或英文。
10. 观察屏幕底部是否自动出现统一浮层，并确认浮层内能实时显示识别草稿和波形。
11. 说完后再按一下右 Alt，等待腾讯云返回最终结果和 DeepSeek 整理结果，确认最终文本进入第 8 步聚焦的输入框。
12. 选中一段现有文字，按右 Alt 后说“翻译”或“总结”，再按右 Alt 结束；确认结果显示在底部小文本框中，而不是写入当前输入框。
13. 在微信、浏览器、邮件或任意可读窗口前，让光标停在回复输入框里，长按右 Alt 约半秒，确认底部显示 `看屏中` / `AI 回复中`，最后把一条回复写入输入框但不自动发送。
14. 再开始一次语音输入或看屏回复，在处理中按 ESC，确认底部浮层显示“已取消”并自动收起，当前输入框不写入任何文字。
15. 点击 `设置快捷键...`，点击 `录制新快捷键`。
16. 按下一个自定义快捷键，例如 `Control + Space`。
17. 按一下新快捷键开始说话，再按一下结束；长按新快捷键触发看屏回复，确认不用点击按钮也能触发两条链路。

## 预期表现

- 未授权时，按快捷键会先引导授权，不会静默失败。
- 按一下快捷键后菜单栏标题会从 `NexVoice` 变为听写/转写状态；再次按下后结束并生成文本。长按快捷键会进入看屏回复，不会开始录音。
- 菜单应显示 `ASR：腾讯云实时 ASR（中英自动）`；不再提供“语言：中文 / English”这种输入语言选择。
- `输出：中文` 会让最终文本以中文为主，但允许保留英文术语、代码、品牌名和自然的中英混合表达；`输出：English` 会让最终文本整理为英文。
- 菜单应提供 `输出模式` 二级菜单；当前四个模式为 `标准模式（默认）`、`社交达人`、`强化嘴替`、`冷静模式`。
- 输出语言和输出模式应相互独立：`输出：中文` 或 `输出：English` 只决定最终语言；`输出模式` 同时决定改写幅度和表达调性。默认 `标准模式` 必须修正吞字、断词、口头禅、错词和断句，同时严格贴合原意、事实、立场和语气强弱。
- 底部波形提示条会自动出现，不需要点击单独的显示窗口按钮。
- 底部浮层应对齐 NexHub 语音输入样式：一个整体深色 blur/tint 胶囊面板，不应出现“文字框 + 波形条”拼接感；声波不应有额外背景框；实时识别草稿使用 13 号字，随内容自动增高，超过最大高度后在浮层内滚动，底部保留 5 根圆角波形反馈。
- 再次按快捷键结束输入后，浮层应从大框缩小成 loading 条，不能直接消失；DeepSeek 整理完成并写入文本后才收起。
- 按 ESC 应取消当前语音输入链路：如果腾讯云最终文本还没进入 DeepSeek，则 DeepSeek 不应启动；如果 DeepSeek 已经在整理，则应取消整理任务；无论处于哪个阶段，最终都不应写入文字。
- 看屏回复只读取当前前台应用可见窗口，不会滚动页面、不会读取屏幕外历史记录，也不会自动点击发送。微信群或多人聊天会根据 OCR 文字位置粗略区分 `我 / 对方 / 未知`，再交给 DeepSeek 判断上下文；复杂气泡布局可能仍需后续优化。
- 浮层出现、波形态到文字态、录音态切到 loading 态、最终收起都应有短促柔和的过渡动画；波形态到文字态应从底部中心稳定展开，文字容器淡入，不应硬切或斜向滑出。
- 实时文字更新时应有轻微透明度渐变，避免逐字出现时过于生硬；如果系统开启“减少动态效果”，动画应自动简化。
- 不说话时，波形应基本保持稳定基线，不应出现持续高频抖动。
- 再次按右 Alt 停止录音后，波形提示条应立即消失；App 会发送腾讯云结束消息并等待最终结果。
- 录音过程中会按约 200ms PCM 音频块实时上传到腾讯云实时 ASR。
- 腾讯云实时 ASR 固定使用 `16k_zh_en`，由 ASR 自动识别中文、英文或中英混合输入。
- 腾讯云配置文件位于 `~/Library/Application Support/NexVoice/TencentCloudASR.json`，文件权限应为 `600`。
- 如果菜单显示 `ASR：腾讯云实时 ASR（缺少 SecretKey）`，说明还没有完成云端配置，按快捷键会失败但不会崩溃。
- DeepSeek 配置文件位于 `~/Library/Application Support/NexVoice/DeepSeek.json`，文件权限应为 `600`。
- 如果完全没有识别到文字，底部浮层会显示短暂的“未识别到语音”轻提示，然后自动收起；这不应弹出腾讯云错误提示，也不应进入吓人的错误态。
- 如果用户口述内容里包含多个要求、步骤、问题、原因或结论，DeepSeek 应把杂乱口语整理成通顺、有逻辑的文本，并在合适时使用 `1. 2. 3.` 编号或分段；短句不应被强行扩写成复杂格式。
- DeepSeek 不应把所有内容都强行结构化。普通聊天、评论、邮件回复、单个观点或连续想法默认应输出自然段；只有任务清单、步骤、方案对比，或用户明确要求“整理成几点/结构化/列出来”时才编号。
- DeepSeek 整理会自动携带上下文包：当前 App、焦点控件、输入框已有内容片段、是否为选中文本指令模式和个人词库。自动风格应基于上下文更稳定地判断普通输入、Agent 协作、即时沟通或邮件回复。
- DeepSeek 超时不再是所有场景固定 5 秒：短普通输入仍以低延迟为主；较长内容和选中文本指令会获得更长等待时间，减少明明已经进入 DeepSeek 但被过早超时回退的情况。
- 如果启动语音前存在选中文本，NexVoice 应将该选中文本作为上下文，将语音识别结果作为指令交给 DeepSeek；结果显示在选区附近的小文本框，不写回输入框。位置优先跟随触发语音时的鼠标位置，捕获选区会临时使用系统剪贴板，读取后应恢复原剪贴板内容。
- 如果选中文本来自可编辑输入框，NexVoice 应视为普通写回目标：结束语音后直接用最终文本覆盖当前选区，不进入“选中文本 + 语音指令”模式。
- 如果 DeepSeek 整理失败或超时，App 应回退写入腾讯云 ASR 原文，不能丢失本次语音输入。
- DeepSeek 诊断日志位于 `~/Library/Application Support/NexVoice/Logs/DeepSeekRewrite.jsonl`。如果出现“没有润色/没有结构化/直接回退原文”，应先检查该日志中的 `event`、`operation`、`latencyMs`、`httpStatus`、`finishReason`、`errorMessage`、`promptPreview` 和 `outputPreview`。
- 看屏回复的 OCR 原文不会完整写入 DeepSeek 诊断日志；`screen_reply` 日志只记录 prompt 字符数和状态，避免把屏幕可见内容长期留在本地日志里。
- 腾讯云 ASR 诊断日志位于 `~/Library/Application Support/NexVoice/Logs/TencentCloudASR.jsonl`。如果出现实时文字慢、没有 final、误报未识别到语音，应先检查该日志中的 `started`、`first_partial`、`finish_requested`、`final`、`no_speech` 和 `failed` 事件。
- 可用 `.build/debug/NexVoiceRewriteEval --dry-run --include-prompt` 跳过 ASR 检查 DeepSeek prompt 是否带上下文；网络可用时运行 `.build/debug/NexVoiceRewriteEval` 批量收集真实 DeepSeek 输出样本。
- 如果命令行环境无法连接 DeepSeek，可以在新版 NexVoice 菜单中点击 `运行 DeepSeek 评测`，让 App 自己的进程直接跑真实样本；报告会写入 `~/Library/Application Support/NexVoice/EvalReports`。
- 系统剪贴板会被临时用于粘贴最终文本；如果粘贴期间用户没有改动剪贴板，NexVoice 会自动恢复原剪贴板内容。
- 如果腾讯云鉴权、连接、发送或识别确实失败，会在同一底部浮层里显示统一的“转写失败”错误态；权限申请等非语音流程仍可使用系统提示。

## 已知限制

- 腾讯云实时 ASR 会上传语音音频到腾讯云；后续如需离线模式，可切回 SenseVoice 本地链路。
- 当前不会把腾讯云临时识别结果写入输入框；只在最终结果到达并完成 DeepSeek 整理后一次性写入，避免输入框文字反复跳动。
- 全局键盘事件和当前输入框写入都依赖辅助功能权限；如果右 Alt 没反应，或有提示条但没有文字进入输入框，点击菜单里的 `申请辅助功能权限` 并重新授权。
- 看屏回复依赖屏幕录制权限；如果长按右 Alt 后提示需要权限，需在系统设置中允许 NexVoice 屏幕录制，并重新打开 App。
- 密码框、部分安全输入框或拒绝系统粘贴事件的 App 可能无法接收语音输入。
- SenseVoice Small 和 WhisperKit large-v3 保留为后续 Benchmark 对照和兜底。
- 实时同传还未完成。
