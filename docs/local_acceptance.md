# NexVoice 本地验收说明

更新时间：2026-06-18

## 当前可验收范围

本轮可验收的是腾讯云实时 ASR 语音输入链路：

1. 打开 NexVoice 菜单栏 App。
2. 申请麦克风权限。
3. 申请辅助功能权限。
4. 选择中文或英文。
5. 打开任意可输入文字的位置，例如备忘录、浏览器搜索框、聊天输入框。
6. 按一下默认右 Alt 快捷键开始语音输入。
7. 在屏幕底部看到自动叠加的小型波形提示条。
8. 再按一下右 Alt 结束，波形条应立即收起；腾讯云返回最终结果后，最终文本直接进入当前聚焦的输入框。
9. 在设置里自定义快捷键并再次触发语音输入。

当前还没有实时同传和最终润色。腾讯云实时 ASR 需要先配置 AppID、SecretId 和 SecretKey；SenseVoice 本地后端保留为兜底和对照。

## 启动方式

在项目根目录运行：

```bash
./scripts/configure_tencent_asr.sh
./scripts/dev_run.sh
```

脚本会重新构建并打开：

```text
dist/NexVoice.app
```

打开后，顶部菜单栏会出现 `NexVoice`。

## 验收步骤

1. 点击菜单栏 `NexVoice`。
2. 点击 `申请麦克风权限`，在系统弹窗中允许。
3. 点击 `申请辅助功能权限`，在系统弹窗或系统设置中允许 NexVoice 控制电脑；这是接收全局键盘事件和把文字输入到其他 App 的必要权限。
4. 如果系统设置列表里看不到 NexVoice，先回到菜单再次点击 `申请辅助功能权限`，让 macOS 先登记这个 App。
5. 如果 macOS 提示需要重新打开 App，退出 NexVoice 后重新运行 `./scripts/dev_run.sh`。
6. 根据要测试的语音点击 `语言：中文` 或 `语言：English`。
7. 打开一个文本输入框，并让光标停在里面。
8. 按一下键盘右 Alt 并说一段中文或英文。
9. 观察屏幕底部是否自动出现小型波形提示条；条内只显示波形，不显示文字。
10. 说完后再按一下右 Alt，等待腾讯云返回最终结果，确认最终文本进入第 7 步聚焦的输入框。
11. 点击 `设置快捷键...`，点击 `录制新快捷键`。
12. 按下一个自定义快捷键，例如 `Control + Space`。
13. 按一下新快捷键开始说话，再按一下结束，确认不用点击按钮也能触发语音输入。

## 预期表现

- 未授权时，按快捷键会先引导授权，不会静默失败。
- 按一下快捷键后菜单栏标题会从 `NexVoice` 变为听写/转写状态；再次按下后结束并生成文本。
- 底部波形提示条会自动出现，不需要点击单独的显示窗口按钮。
- 底部波形提示条只显示麦克风波形，不显示实时草稿、输入状态文字或最终结果；视觉应是深色胶囊 + 细密竖条。
- 不说话时，波形应基本保持稳定基线，不应出现持续高频抖动。
- 再次按右 Alt 停止录音后，波形提示条应立即消失；App 会发送腾讯云结束消息并等待最终结果。
- 录音过程中会按约 200ms PCM 音频块实时上传到腾讯云实时 ASR。
- 腾讯云配置文件位于 `~/Library/Application Support/NexVoice/TencentCloudASR.json`，文件权限应为 `600`。
- 如果菜单显示 `ASR：腾讯云实时 ASR（缺少 SecretKey）`，说明还没有完成云端配置，按快捷键会失败但不会崩溃。
- 如果完全没有识别到文字，底部波形提示条会自动收起，菜单栏状态会短暂进入出错状态。
- 系统剪贴板会被临时用于粘贴最终文本；如果粘贴期间用户没有改动剪贴板，NexVoice 会自动恢复原剪贴板内容。
- 如果腾讯云鉴权、连接、发送或识别失败，会通过系统提示或菜单栏状态提示错误，不会在波形提示条内显示文字。

## 已知限制

- 腾讯云实时 ASR 会上传语音音频到腾讯云；后续如需离线模式，可切回 SenseVoice 本地链路。
- 当前不会把腾讯云临时识别结果写入输入框；只在最终结果到达后一次性写入，避免输入框文字反复跳动。
- 全局键盘事件和当前输入框写入都依赖辅助功能权限；如果右 Alt 没反应，或有提示条但没有文字进入输入框，点击菜单里的 `申请辅助功能权限` 并重新授权。
- 密码框、部分安全输入框或拒绝系统粘贴事件的 App 可能无法接收语音输入。
- SenseVoice Small 和 WhisperKit large-v3 保留为后续 Benchmark 对照和兜底。
- 同传、DeepSeek 最终润色还未完成。
